# Calendar Slots (`@tinycld/calendar-slots`)

Let users book a slot on your calendar. Organizations create public booking pages; external visitors pick a time and book without authentication. Bookings auto-create calendar events and notify the page owner.

## Database collections

| Collection | Table | Key fields |
|---|---|---|
| `booking_pages` | `pbc_book_pages_01` | org, owner, slug (unique), name, intro_text, active |
| `booking_slot_types` | `pbc_book_slots_01` | page (→ booking_pages, cascade delete), name, duration_minutes, padding_minutes, color |
| `booking_availability` | `pbc_book_avail_01` | page (→ booking_pages, cascade delete), day_of_week (0-6), start_time, end_time |
| `bookings` | `pbc_book_bookings_01` | page, slot_type, calendar (→ calendar_calendars), guest_name, guest_email, start, end, status (confirmed/cancelled), calendar_event |

Access rules: `booking_pages` viewable by org members, manageable by owner. `booking_slot_types` and `booking_availability` restricted to page owner. `bookings` viewable by page owner, **creatable by anyone** (public) — security is enforced server-side in the API handler.

## How it works

### Admin flow (authenticated)
1. Org member creates a booking page (name, slug) via `screens/[id].tsx`
2. Adds slot types (e.g. "Quick Chat", "Meeting") with duration/padding via `SlotTypeEditor`
3. Defines weekly availability (e.g. Mon-Fri 9-5) via `AvailabilityEditor`
4.  The page becomes active; public URL is `/p/calendar-slots/{orgSlug}/{slug}`

### Public booking flow (unauthenticated)
1. Visitor opens `/p/calendar-slots/{orgSlug}/{slug}` → `public-screens/[orgSlug]/[slug].tsx`
2. Fetches `GET /api/book/{orgSlug}/{slug}/available-slots` → shows available time slots grouped by date
3. Visitor picks a slot, enters name + email
4. Submits via `POST /api/book/{orgSlug}/{slug}/book` → creates a `bookings` record
5. Server hook triggers auto-creation of a calendar event on owner's calendar
6. Owner receives a notification

### Server-side slot generation (`register.go`)
- `GET /api/book/{orgSlug}/{slug}/available-slots`: loads org + page, slot types, availability windows, and owner's calendar. Collects busy ranges (calendar events + existing bookings). Generates slots at duration+padding intervals for the next 28 days, skipping busy overlaps and past times.
- `POST /api/book/{orgSlug}/{slug}/book`: validates input, checks for conflicts, creates booking record with `confirmed` status.
- `OnRecordAfterCreateSuccess`: goroutine that creates a calendar event with iCal UID and sends notification to the page owner.

## Build & tooling

- **Dev:** Run from the workspace root via `pnpm --filter app run dev` (builds Go server, starts PocketBase, Expo, and SSL proxy at localhost:7100)
- **Typecheck:** `pnpm run typecheck` → `tinycld-pkg typecheck`
- **Tests:** `pnpm run test` (unit) / `pnpm run test:e2e` (Playwright) → `tinycld-pkg`
- **Lint:** Run from `app/` via `pnpm run lint` (biome)
- **Go build:** The package's `server/` directory is compiled as part of the app's Go server via `app/server/go.work`. The member's `go.work` (generated by `app/scripts/gen-server.ts`) allows standalone builds.
- **Route generation:** `app/scripts/generate.ts` creates Expo Router re-export files. Admin screens go under `app/app/a/[orgSlug]/calendar-slots/`, public screens go under `app/app/p/calendar-slots/[orgSlug]/[slug].tsx`.

## Key patterns

- **Mutations:** Uses `@tinycld/core/lib/mutations` with generator-function `yield` pattern for all CRUD
- **Queries:** `useOrgLiveQuery` with `@tanstack/db` query builder, scoped to current org
- **Forms:** `react-hook-form` with `@tinycld/core` themed input components
- **Navigation:** `useOrgHref` for org-scoped paths, `expo-router` Link/useRouter
- **Theming:** `useThemeColor` for color tokens that work in light and dark mode
